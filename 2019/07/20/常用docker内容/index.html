就是普通的常用docker内容

安装最新的 Docker Community Edition 程序包:
sudo yum install -y docker
启动 Docker 服务:
sudo service docker start
将 ec2-user 添加到 docker 组，以便您能够执行 Docker 命令，而无需使用 sudo:
sudo usermod -a -G docker ec2-user
退出，再重新登录以接受新的 Docker 组权限。您的新 SSH 会话将具有相应的 Docker 组权限，
而无需再使用 sudo。
为了避免每次使用Docker命令时都需要切换到root账户，可以将当前用户加入安装中自动创建的 docker用户组:
sudo usermod -aG docker $USER

当然 更简单的方式是
sudo -s

因为电脑是win7 所以下载Docker Toolbox Win10或mac有其他的安装程序
然后安装，
安装完毕后，使用管理员权限运行。 如果报错，vmanage.box之类的问题，卸载自带的visual box，因为这个版本太高，会报错
安装visual box4.6版本。 完毕后，重新运行程序。
进入后，是win命令行。 如果想要使用linux命令行， 执行:docker-machine ssh default 推出linux命令行 执行:exit
输入
docker pull hello-world docker run hello-world

如果有一个repository，去build那里的Dockerfile或者，直接pull那的image
需要 docker login –username=用户名 –password=密码 要登录的地址
然后执行 docker pull image地址
如果，是本地的Dockerfile，文件里的FROM是从一个repository中做base，则也需要登录。
build本地file的命令是:
docker build -t 起名字:版本号 .

如果，build或者pull的时候报错， EOF，这种错误很常见， 一般来说都是网速太慢导致的。
可以有以下几种方式:
1.添加镜像来加速
2.增加同时下载的线程数
在linux命令行下:
vim /etc/docker/daemon.json
{ “registry-mirrors”: [“https://registry.docker-cn.com"], “insecure-registries”: [“registry.docker-cn.com”], “max-concurrent-downloads”: 10,
“log-driver”: “json-file”,
“log-level”: “warn”,
“log-opts”: {
“max-size”: “10m”,
“max-file”: “3” }
}

修改daemon文件后，要重启docker才能生效
Linux: service docker restart Win: docker-machine restart
docker镜像保存在 /var/lib/docker/ 目录下
ref地址：https://docs.docker.com/registry/spec/api/
虽然这里面写着Docker支持断点续传
但实际上不支持，如果一段时间下不完，就重新从头开始再下一遍。
所以，修改daemon文件是目前唯一可以加速的方式。

后台执行容器
docker run -d tail -f /dev/null
进入执行中的容器
docker exec -it CONTAINER_ID bash
